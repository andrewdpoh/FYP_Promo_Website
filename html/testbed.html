Elastic Configurations

The Elastic Stack, also known as the ELK Stack, is a platform comprised of Elasticsearch, Kibana, Beats and Logstash. It enables users to take data from many sources and formats to perform searching, analysing and visualisation of data in real time. For more information, visit Elastic's website. In this project, we primarily focus on Elastic Security which is a feature of the Elastic Stack. Elastic Security combines Security Information and Event Management (SIEM) threat detection features with Endpoint Detection and Repsonse (EDR) capabilities into one solution. Below briefly explains the configurations we put in place in our Elastic setup.

Policies

A policy was created which was applied to all Elastic Agents. There is 1 agent installed on Windows 11A and another in Windows 11B. Policies allows for configurations such as integrations and rules to be applied to all the Agents at once. We also enabled the Malware Protection setting in this policy.

Integrations

Elastic Defend Integration: Provides prevention, detection and response capabilities on the host machine. This allows for deep visibility into the host and quick response from the Elastic Stack.
Windows Integration: Allows for the monitoring of the Windows OS, services, applications and more. It helps in the collection of metrics and logs from the host machine.

Rules

Pre-built Detection Rules: Enabled all pre-built detection rules with the "os:Windows" tag and updated the rule-schedule to run every 1 minute. The repository for these rules can be found on GitHub.
Malware Detection Rules: These rules are part of the Malware Protection setting in the agent policy. The repository for these rules can also be found on GitHub.
