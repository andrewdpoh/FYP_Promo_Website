What is Bamboo?

Bamboo is a Command-and-Control (C2) Framework that applies tested evasion techniques to exploits, allowing your attacks to bypass detection from Elastic Stack.

Features

Evasion

3 undetected shellcode injection techniques and an EDR blinding tool

Exploits

5 pre-loaded exploits to test, along with the option to add your own

Post-Exploitation

2 useful post-exploitation tools which evade detection

User & Agent Management

Users and agents can be added and removed with ease using the GUI

Quick & Simple Operation

Add your exploits, start the agent and use any one of Bamboo's evasion techniques with ease

Encrypted Communication

Communication between Teamserver and Agent are via HTTPS and Websocket Secure

Project Scope

Problem Statement

The widespread use of powerful threat detection tools such as SIEMs and EDRs forces penetration testers to invest significant time and effort into discovering and implementing evasion techniques to bypass detection rules, prolonging the testing process.

Solution

A tool that contains evasion techniques which allow penetration testers to perform attacks while evading detection from the Elastic Stack.

Assumptions

Item Description Reason
Target OS Windows 11 This is the latest version of the Windows OS which is commonly used in enterprises
Detection to Evade The Elastic Stack (Basic License) will be used. The Elastic Endpoint Defender will also replace Windows Defender as the target's AV The Elastic Stack is widely used by enterprises to collect logs and detect threats due to its open-source and cost-effective options. The Basic License will be used due to budget constraints.
Type of Exploits Local Privilege Escalation (LPE) exploits for Third-party applications The project scope is for single-host exploitation, thus the most valuable exploits are LPEs. We also only target third-party applications as they are less likely to be automatically patched than Windows vulnerabilities.
